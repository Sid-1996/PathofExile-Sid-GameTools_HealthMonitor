# 🛠️ 本地開發便利工具

> **⚠️ 重要區分**：此檔案僅供本地開發使用，**不**會發布到開源倉庫

## 🎯 目的

提供開發者本地工作時的便利工具和腳本，提升開發效率，但不影響開源專案的純潔性。

## 📁 本地專用檔案

### 🛠️ 自動化腳本（本地專用）
- `scripts/cleanup.bat` - 清理建構檔案
- `scripts/release.bat` - 快速發布流程
- `scripts/secure-release.bat` - 安全發布流程

### 🔧 開發輔助
- `.vscode/` - VS Code 設定
- `.venv/` - 本地虛擬環境
- `build/` - 建構暫存（可清理）
- `dist/` - 本地測試包（可清理）

### 📝 開發筆記
- `.windsurf/workflows/` - AI 技能文檔
- 開發時的臨時筆記和測試檔案

## 🚫 不應開源的內容

### 🔒 敏感資訊
- 個人開發環境設定
- 本地測試資料
- 開發時的臨時密鑰或配置

### 🛠️ 便利性工具
- 自動化腳本（可能包含個人偏好）
- IDE 設定檔案
- 建構優化設定

### 📊 開發數據
- 測試結果和日誌
- 性能分析數據
- 開發時間統計

## 🔄 使用原則

### ✅ 開發階段
1. **使用便利工具**：充分利用本地腳本提升效率
2. **記錄改進**：在開發筆記中記錄優化
3. **測試驗證**：使用本地工具快速測試

### 🚀 發布前
1. **清理敏感資訊**：確保不包含個人資料
2. **檢查開源內容**：只發布核心程式碼和文檔
3. **使用標準流程**：通過安全發布腳本

## 📋 本地 vs 開源檔案清單

| 類型 | 本地專用 | 開源發布 | 範例 |
|------|---------|---------|------|
| **腳本** | ✅ `scripts/release.bat` | ❌ | 便利發布工具 |
| **配置** | ✅ `.vscode/settings.json` | ❌ | 個人 IDE 設定 |
| **環境** | ✅ `.venv/` | ❌ | 本地虛擬環境 |
| **文檔** | ❌ `.windsurf/workflows/` | ✅ `README.md` | AI 技能文檔 |
| **核心** | ❌ `src/health_monitor.py` | ✅ `src/health_monitor.py` | 主要程式碼 |
| **建構** | ✅ `build/` | ❌ | 建構暫存 |
| **發布** | ✅ `dist/` | ✅ `dist/*.zip` | 最終產物 |
