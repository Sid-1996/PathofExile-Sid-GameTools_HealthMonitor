# 檔案用途與建議流程

## 你的三種工作模式

1. 原始 Python 直接跑  
   - 雙擊：`scripts/run_monitor.bat`
   - 主要程式：`src/health_monitor.py`

2. 一鍵打包 EXE  
   - 雙擊：`scripts/build_exe.bat`
   - 打包腳本：`tools/build.py`
   - 輸出位置（預期）：`dist/GameTools_Package/GameTools_HealthMonitor.exe`

3. 打包完直接測試 EXE  
   - 雙擊：`scripts/test_built_exe.bat`

## 目錄用途（重點版）

- `src/`  
  日常開發主目錄（建議只改這裡）。

- `src for DEVELOPER/`  
  目前打包流程相容目錄。`build_exe.bat` 會先把 `src/` 內核心檔同步過來再打包。

- `scripts/`  
  一鍵操作入口（安裝依賴、跑原始碼、打包、測試 EXE）。

- `tools/`  
  打包程式與工具。

- `docs/`  
  文件說明。

## 建議固定流程

1. 先安裝依賴：`scripts/install_dependencies.bat`
2. 開發與測試原始碼：`scripts/run_monitor.bat`
3. 需要出包時：`scripts/build_exe.bat`
4. 出包後驗證：`scripts/test_built_exe.bat`

## 目前你最該避免的事

- 不要再用「整批刪資料夾」方式整理。
- 不要直接刪 `src/`、`src for DEVELOPER/`、`scripts/`、`tools/`。
- 每次大改結構前先 commit，一步一步改。
